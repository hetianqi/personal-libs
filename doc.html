概述

本文将目前项目中常用的控件列表列出进行API说明，后续也会持续更新...

由于这些控件都是基于bootstrap进行扩展的，因此在页面中需要先引入bootstrap相关的css和js文件，针对IE8，还需要引入html5shiv.min.js和respond.min.js文件。 页面中引入以上文件的代码如下： 

							
<head>
    <meta charset="utf-8">
	<meta http-equiv="X-UA-Compatible" content="IE=edge">
	<meta name="viewport" content="width=device-width,minimum-scale=1.0,maximum-scale=1.0,user-scalable=no">
	<title>Title</title>
	<link rel="stylesheet" type="text/css" href="./assets/css/bootstrap.min.css">
	<link rel="stylesheet" type="text/css" href="./assets/css/common.css">
	<link rel="stylesheet" type="text/css" href="./assets/css/plugins.css">
	<link rel="stylesheet" type="text/css" href="./assets/css/tlayer.css">
	<!--[if lt IE 9]>
		<script src="./assets/js/html5shiv.min.js"></script>
		<script src="./assets/js/respond.min.js"></script>
	<![endif]-->
	<script type="text/javascript" src="./assets/js/jquery-1.11.3.js"></script>
	<script type="text/javascript" src="./assets/js/bootstrap.min.js"></script>
	<script type="text/javascript" src="./assets/js/jquery.util.js"></script>
	<script type="text/javascript" src="./assets/js/jquery.plugins.js"></script>
	<script type="text/javascript" src="./assets/js/jquery.uploadify.js"></script>
	<script type="text/javascript" src="./assets/js/jquery.tlayer.js"></script>
</head>

其中，common.css是针对bootstrap进行reset以及对一些公用样式进行封装的样式文件；tlayer.css和jquery.tlayer.js是弹出框、提示框和加载框插件；jquery.uploadify.js是上传插件（样式文件在plugins.css中）；plugins.css和jquery.plugins.js是常用的插件集合（包括ajax分页、快速生成表格、下拉框美化、滚动条美化等）；util.js文件对常用函数（如获取url中的键值对、获取浏览器版本信息等）进行了封装。使用者可以按需引入或全部引入。

下面将对除bootstrap自带以外的控件进行说明。

引言

关于目录结构

前端代码推荐放到assets目录下，目录结构为： 

							
assets
	|-- css 		存放css文件
	|-- fonts 		存放web字体文件
	|-- img 		存放图片
	|-- js 			存放js文件
	|   |-- app		存放页面逻辑相关的js文件
	|   |-- lib 	存放jquery,requirejs等js库文件
	|-- plugins 	存放其他的整体的组件

关于引入方式

js文件推荐采用AMD的方式引入，既方便管理，也可以通过异步加载js文件减少页面加载时间，配置如下： 

							
require.config({
    baseUrl: './assets/js',
    paths: {
        'jquery': 'lib/jquery-1.11.3.min',
        'bootstrap': 'lib/bootstrap.min',
        'uploadify': 'lib/jquery.uploadify',
        'tlayer': 'lib/jquery.tlayer',
        'plugins': 'lib/jquery.plugins',
        'util': 'lib/util',
        'selector': '../plugins/selector/selector'
    },
    shim: {
        'bootstrap': {
            deps: ['jquery']
        },
        'uploadify': {
            deps: ['jquery']
        },
        'tlayer': {
            deps: ['jquery']
        },
        'plugins': {
            deps: ['jquery']
        },
        'selector': {
            deps: ['jquery', 'tlayer']
        }
    },
    urlArgs: 'bust=' + (new Date()).getTime()   //开发环境下禁用缓存，生产环境要移除
});

表单正则验证提示

表单正则验证提示组件提供在输入框失去焦点时进行正则验证，并把错误信息提示给用户。为输入框添加data-uitype="validate"属性即可使用该功能。 

							
<input type="text" id="example-validate" data-uitype="validate" data-reg="/^\d[6-16]$/" data-msg="请输入6-16位数字！">

选项

表单正则验证提示组件接收一个对象参数，详细配置如下所示： 

							
{
    reg         : '',   // 正则表达式，优先级大于type
    type        : '',   // 内置正则表达式email, idcard, tel, mobile
    msg         : '',   // 错误提示信息
    placement   : ''    // 提示框放置位置，默认自动计算
}

文本框自动补全

文本框自动补全组件提供在输入过程中需要根据关键字进行搜索，然后将结果展示在待选列表中的功能。 

							
<input type="text" id="example-autoComolete">
							
							
$('#example-autoComolete').autoComplete({
	async: {
        url: 'http://10.0.0.5:8011/Selector/Product/SearchProduct',
        dataType: 'jsonp',
        dataField: null,
        searchField: 'keyword'
    },
    template: '<td>#{zhengjiBm}</td><td>#{zhengjiName}</td>',
    maxNum: 10,
    callback: function (data) {
    	$('#autoComplete').val(data.zhengjiName);
    }
});

选项

文本框自动补全接收一个对象参数，详细配置如下所示： 

							
{
    async: {                        //远程请求获取数据参数，和ajax请求参数基本一致
        url: '',                    //远程请求url
        type: 'GET',                //请求方式
        data: false,                //请求入参，不包括搜索关键字，搜索关键字会自动带入
        dataType: false,            //返回数据类型，支持jsonp
        dataField: 'data',          //返回数据的字段中那个字段表示数据列表，null表示返回数据即数据列表
        searchField: 'keyword'      //搜索关键字名称
    },
    dataList: [],                   //数据列表，支持本地数据列表
    localSearchField: null,         //本地搜索字段
    template: '<td>#{}</td>', 	//列表模板
    listWidth: false,               //列表宽度
    maxNum: null,                   //最大显示条数
    autoHide: false,                //列表是否自动在3秒后隐藏
    callback: false,                //选中数据之后的回掉，参数为选中的数据
    onInit: false                   //组件初始化回调
}

下拉选择框

为普通的select标签添加data-uitype="select"属性可将其美化，使各个浏览器表现统一，并提供更多选项。 

							
<div id="multi-ui-select"></div>
							
							
$('#multi-ui-select').uiSelect({
	dataList: ['中国', '美国', '日本', '英国', '法国', '俄罗斯', '印度', '巴西'],
	name: 'test_multi_select',
	multi: true
});

参数

详细配置参数如下所示，可以将选项通过data属性或 JavaScript 代码传递。对于 data 属性，需要将参数名称放到data-之后，例如data-searchbox="1"

							
{
    name            : '',               // 作为表单的name
    dataList        : [],               // 数据项
    template        : '#{}',   // 模板
    textField       : '',               // text字段名
    valueField      : '',               // value字段名
    seprator        : ',',              // 多选值分隔符
    disabled        : false,            // 是否禁用
    searchbox       : false,            // 是否显示搜索框
    multi           : false             // 是否多选
}

方法

setValue(value)

为下拉框设置选中项，接收一个字符串（单选）或数组（多选）参数。

setDisabled(disabled)

设置下拉框的禁用状态，接收一个bool值参数，true表示禁用，false为启用。

destroy()

销毁下拉框，并还原select标签

弹出框

弹出框以对话的形式出现，通过某些事件（如click）触发，具有多种类型和实用的功能。 

							
$.msg('这是消息框');

$.tips('这是提示框');

$.loading('这是加载框');

$.alert('这是警告框');

$.confirm('这是询问框', function (result) {
	
});

$.content({
	header: '内容框',
	content: {
		html: '这是内容框'
	},
	footer: [{
		text: '保存',
		style: 'primary',
		callback: function () {

		}
	}, {
		text: '取消'
	}]
});

种类

所有的弹出框都是基于content内容框改造的。

消息框 $.msg(msg, fn)

消息框接收两个参数。第一个参数必选，为字符串，表示消息文本；第二个参数可选，为函数，表示消息框关闭后回调的函数。

提示框 $.tips(msg, type, fn)

提示框接收三个参数。第一个参数必选，为字符串，表示消息文本；第二个参数可选，如果为函数，则表示提示框关闭后回调的函数，此时消息类型type参数值为1；第三个参数如果存在，则为提示框关闭后的毁掉参数，此时第二个参数表示提示消息类型，可选值有0，1，2，3，对应图标 的位置。

加载框 $.loading(msg)

加载框接收一个必选参数，为字符串，表示消息文本。

警告框 $.alert(title, msg, callback)

警告框接收三个参数。第一个参数必选，为字符串，表示警告框标题；第二个参数可选，如果为字符串，则为消息内容，如果为函数，则表示用户点击确定之后的回调函数，此时第一个参数为消息内容；第三个参数可选，为函数，表示用户点击确定之后的回调函数。

询问框 $.confirm(title, msg, callback)

询问框参数与警告框相同。

内容框 $.content(options)

内容框接收一个对象。详细配置为：

    duration        : 150,          //显示和隐藏的时间
    win             : window,       //窗口上下文对象
    onEsc           : false,        //当弹出框触发Esc按钮时执行的回调函数
    auto            : true,         //默认自动显示，否则使用$.tlayer("show", layerID);
    theme           : 'blue',       //主题，目前提供default，blue, black三种

    content         : {             //弹出框内容区域的设置  
                                    //当src, url和html属性同时存在，优先级依次降低

    },        

    footer          : false,         //弹出框的底部参数设置。值为false时表示弹出框无底部
    /**
     * 如果footer存在，可按照以下格式设置相关参数
     *
     * [         //底部按钮设置
     *     {
     *         text         : false,    //按钮文字
     *         buttonID     : false,    //按钮的id，可用于用户自定义设置按钮的样式
     *         style        : 'default',       //按钮样式
     *         callback     : false,    //按钮点击时执行的操作
     *     }
     * ]
     * 
     */
    
    onInit         : false,            //初始化完成回调

}

方法

关闭弹出框 $.tlayer('close', layerID)

close方法接收一个参数，layerID表示弹出框的id，也可以传入弹出框对象

文件上传

文件上传是在uplpoadify插件的基础上进行改造的，主要添加符合我们自己业务的上传模板以及对一些样式进行改造。 

							
<button id="example-uploadify" class="btn btn-sm btn-primary"></button>
							
							
$('#example-uploadify').uploadify();

选项

上传组件提供两种类型，一种是基于flash的封装，用于在IE10以下使用，另一种是基于HTML5原生的封装，使用者自己判断引入哪个文件。两者都接收一系列可选参数进行配置，如下所示： 
							
{
	id                  : '',           // 组件id
    uploader            : '',           // 服务器接收地址
    uploadDesc          : '',           // 上传描述
    auto                : true,         // 是否自动上传
    multi               : true,         // 是否多选
    method              : 'POST',       // 自定义参数发送方式
    formData            : {},           // 自定义参数
    fileObjName         : 'FileData',   // 文件发送到服务器端时的名称
    defaultTemplate     : true,         // 是否采用默认模板
    fileTypeExts        : '',           // 文件类型限制
    fileSizeLimit       : '2048 MB',    // 文件大小限制
    onSelect            : false,        // 选择文件回调，每一个文件回调一次
    onQueueComplete     : false,        // 队列完成回调
    onUploadStart       : false,        // 上传开始回调
    onUploadProgress    : false,        // 上传中回调
    onUploadSuccess     : false,        // 上传成功回调
    onUploadError       : false,        // 上传出错回调
    onUploadComplete    : false         // 上传完成回调
}

自动化表格

自动化表格插件将重用的表格功能进行封装，提供固定表头滚动、表头点击排序、单元格编辑、列宽度拖动、树形表格等功能。 

							
<div id="example-table"></div>
							
													
$("#example-table").table({
	url: "https://api.douban.com/v2/movie/top250?start=1",
	crossDomain: true,
	data: function () {
		return {
			keyword: $('#keyword').val()
		};
	},
	indexField: "start",
	sizeField: "count",
	dataField: "subjects",
	height: 400,
	pageSize: 20,
	colOptions: [{
		name: 'ID',
		field: 'id',
		edit: {
			replace: function (rowData) {
				return '<input value="'+rowData.id+'" />';
			}
		},
		sort: {
			sname: 'id',
			sorder: 'asc'
		},
		handler: function (value, data) {
			return '<a href="'+data.alt+'" target="_blank">'+data.id+'</a>';
		}
	}, {
		name: '名称',
		field: 'title'
	}, {
		name: '原始名称',
		field: 'original_title'
	}, {
		name: '年代',
		field: 'year'
	}, {
		name: '类型',
		field: 'genres'
	}, {
		name: '收藏数',
		field: 'collect_count'
	}],
	tableClass: "table-hover"
});

    align           : false,                    //全局设置对齐方式

    /*
     * colOptions格式：[{
     *     name: 'ID',                          //列显示名称

     *         callback: function () {},        //编辑回调函数
     *     },

宽度拖动

为一个容器使用添加data-uitype="resize"属性可动态改变容器的宽度。 

							
<div data-uitype="resize" style="width: 400px; height: 400px; border: 1px solid #ddd;"></div>

选项

由于宽度拖动一般伴随着回调函数的绑定，因此建议使用javascript调用的方式，$().uiResize(options)详细配置参数如下： 
							
{
    vertical        : true,                         //拖动方向，true为垂直方向
    border          : 1,                            //可拖动区域的宽度
    minWidth        : 0,                            //容器可改变的最小宽度
    minHeight       : 0,                            //容器可改变的最小高度
    maxWidth        : Infinity,                     //容器可改变的最大宽度
    maxHeight       : Infinity,                     //容器可改变的最大高度
    onResizeStart   : function (e) {},              //开始拖动的回调，返回false的不进行拖动
    onResize        : function (e, distance) {},    //拖动中的回调
    onResizeEnd     : function (e) {}               //拖动结束的回调
}

右键菜单

功能如命。 

							
// 实例化右键菜单
var rMenu = $.rightMenu({
	menu: [{
		icon: '<i class="glyphicon glyphicon-plus" style="color: green;"></i>',
		text: '新建目录',
		callback: function (e, obj) {
			console.log('新建目录');
		}
	}, {
		icon: '<i class="glyphicon glyphicon-remove" style="color: red;"></i>',
		text: '删除目录',
		callback: function (e) {
			console.log('删除目录');
		}
	}, {
		icon: '<i class="glyphicon glyphicon-upload" style="color: blue;"></i>',
		text: '上传文件',
		callback: function (e) {
			console.log('上传文件');
		}
	}]
});

// 显示菜单
$('#table').on('contextmenu', '.table-body .table-tr', function (e) {
	rMenu.show(e.clientX, e.clientY, this);		
	e.preventDefault();
});

选项

详细配置参数如下： 
							
{
    width: 100,                     // 宽度
    autoHide : false,               // 是否自动隐藏
    offsetLeft: 0,                  // 点击处偏移宽度
    offsetTop: 0,                   // 点击处偏移高度
    menu: [{                        // 菜单项选项
        id: '',                     // 菜单id
        icon: '',                   // 菜单图标
        text: '',                   // 菜单文本
        callback: function () {     // 菜单回调，参数为show方法的第三个参数开始的参数

        }
    }]
}

方法

显示菜单 show(x, y, arg1, arg2, ...)

在指定位置显示菜单，并传入回调函数的参数。
show方法的第三个参数起到最后的参数会被传入menu.callback中
